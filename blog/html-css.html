Block-level vs. Inline Elements


All HTML elements are comprimised of an HTML tag and its contents to produce an HTML element. HTML starting tags can also have attributes assigned to them allowing for greater targeting and styling of the element using CSS selectors. HTML elements are classified as block-level or inline elements and each has default display properties that can be specified using CSS.


Examples of the default display properties for all w3c HTML tags have been tested here.

display: inline-block


Using display: inline-block elements gives you the ability to specify width, height, margin and padding values on the element and avoid issues like line-breaks and overlapping. Using a span tag nested inside a paragraph will cause all surrounding content to be pushed away from the box. Another useful feature of display: inline-blocks is that the elements can be positioned alongside each other.


display: inline-blocks are typically used when creating horizontal navigation menu links.

display: inline


An example of an element that has a default display: inline property is the span element. A limitation of the display: inline element is that it will disregard any width and height values that you might have specified in the element's CSS properties, while respecting margin, padding, and border values. This can cause any user inputted text based entries to fall outside the element's predefined parameters in the space designated for your content.
